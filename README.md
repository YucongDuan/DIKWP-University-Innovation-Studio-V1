# DIKWP University Innovation Studio V1

A standalone, offline-first reference prototype for transforming Chinese university education from knowledge transmission alone toward knowledge-grounded innovation capability development.

## Core decision

The system does not replace knowledge teaching with unstructured projects. It configures a course-specific blend of:

1. explicit knowledge foundation
2. active practice
3. guided inquiry
4. authentic project / experiment
5. reflection, defense, and transfer

## Main workflow

Institution and course diagnosis → Course Transformation Canvas → DIKWP learning-to-innovation loop → teacher scaffolds → student studio → Challenge Bank → Innovation Evidence Ledger → project and experiment milestones → authentic assessment and defense → AI-use transparency → equity and workload governance → Course Innovation Passport.

## Quick start

Open `index.html` in a modern browser. No server, database, model API, account, or network is required.

Optional CLI:

```bash
python run_demo.py examples/sample_course.json --out outputs
```

## Non-negotiable boundaries

Do not use this system to automatically:
- admit, exclude, rank, punish, discipline, or award degrees to students
- make high-stakes grades without teacher review
- diagnose disability, mental health, or “innovation potential”
- conduct biometric or emotion-based proctoring
- create covert learner profiles
- evaluate teachers through opaque platform metrics

Use it as a pre-decision evidence and course-design layer. Human teachers, students, curriculum committees, and institutional governance remain responsible.

## Design lineage

This prototype extends `YucongDuan/DIKWP-LearnPath-OS` from personalized learning support toward university course transformation, authentic innovation assessment, teacher workload governance, and institution-level portfolio decisions.
